# Competition compliance

How this project stays within the rules of the **PTCG AI Battle Challenge** for
both categories. The short version: all the heavy machinery (AlphaZero self-play,
the league, PPO) is **offline training that is not part of any submission**, and
the **submitted agent runs offline and within the per-match time limit**.

---

## Simulation category (leaderboard agent)

The submission is a `.tar.gz` with `main.py` + `deck.csv` at the top level plus
the competition `cg/` library; Kaggle Environments calls `agent(observation,
configuration)` each decision point. Build it with **Competition → Export for
Kaggle** (`competition/export.py`).

| Rule | How we comply |
|------|---------------|
| **Self-contained `main.py` + `deck.csv` at top level** | `export.py::build_sim_bundle` emits exactly that; `README` in the archive shows `tar -czvf submission.tar.gz *` after dropping in `cg/`. |
| **Runs offline (no network during matches)** | The shipped `main.py` imports only `csv`/`os`/`random` — no network, no torch. If you wire the engine agents in, the **only** component that can use the network is the `Coach`, and it makes a call **only** when `ANTHROPIC_API_KEY` is set. Kaggle has no such key, and setting **`PTCG_OFFLINE=1`** hard-disables the call regardless. Put `os.environ["PTCG_OFFLINE"] = "1"` at the top of your submission `main.py` for a provable guarantee. |
| **Per-match time limit (~10 min)** | The shipped `main.py` is trivial and fast. For engine agents: the default order is `rl → ismcts → heuristic` (fast, imperfect-information-safe); search agents accept a per-move wall-clock cap via **`PTCG_MOVE_BUDGET`** (seconds), and the Closer/AlphaZero also have explicit node/simulation budgets. Keep `--sims` modest in a submission. |
| **Imperfect information** | The serving agents never read hidden state outside the observation seam; `rl`/`ismcts` are designed for hidden information. The two `decode_observation`/`encode_action` functions are the only thing to bind to the official `cg` schema. |
| **No banned external data / models** | The learned net is trained purely on **engine-generated self-play** (see below). No scraped or third-party datasets, no pretrained external weights. |

### Quick submission checklist
1. Export from the app (or `python -m competition.export`).
2. Add `PTCG_OFFLINE=1` at the top of `main.py` if you wire in engine agents.
3. If you use a search agent, set `PTCG_MOVE_BUDGET` (e.g. `8`) and a modest sim count.
4. Drop in `cg/`, then `tar -czvf submission.tar.gz *`.

## Training tools are *not* part of the submission

`rl/train.py` (PPO), `rl/alphazero_train.py` (AlphaZero self-play, with the
`--league` exploiter population), and the snapshot checkpoints are **offline
training utilities**. They run on your own machine, produce a checkpoint
(`policy_latest.pt`), and never ship inside a submission. The competition places
no restriction on how or where you train — only on the submitted agent's runtime
behaviour (offline + time-bounded), which is covered above. The league plays only
the engine's own agents and frozen snapshots of the learner; no external services.

## Strategy category (writeup)

| Rule | How we comply |
|------|---------------|
| **≤ 2000 words** | `export.py::build_strategy_writeup` targets a sub-2000-word Markdown report; check the count before submitting. |
| **Media Gallery uses only license-compliant Pokémon Elements** (violations = disqualification) | Use **only** the official, license-compliant Pokémon Elements provided by the competition for any imagery. **Do not** paste extracted card art from the bundled PDF/PNGs into the gallery — those are reference data for the local catalog, not redistribution-cleared media. The app's own UI/diagrams (boards, rating curves, agent explanations) are original and safe to screenshot. |
| **Report substance (Model / Deck / Report)** | The app supplies the artifacts: agent designs + rationales, deck metadata, tournament Wilson-CI diagnostics, and the training curve. |

## LLM (Anthropic) usage

The optional `Coach` / `Coach + search` agents call the Anthropic API **only when
you provide your own `ANTHROPIC_API_KEY`, and only when running locally** (e.g. in
the app on your Mac). They are **never** networked inside a competition submission
(`PTCG_OFFLINE=1` and the absence of a Kaggle key both prevent it); offline they
fall back to a heuristic / full one-ply search. This keeps the submission rules
satisfied while still letting you use the LLM-advised agents at home.
